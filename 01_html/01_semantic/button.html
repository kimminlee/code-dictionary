● HTML / Semantic

<button>

사용자 동작을 트리거하는 클릭 가능한 요소

✦ 언제 쓰는가

🖱️
클릭으로 JS 동작을 실행할 때

모달 열기, 데이터 제출, 필터 적용 등 JavaScript 이벤트를 연결해야 하는 모든 상황. <div onclick> 대신 반드시 <button> 을 써야 키보드 접근성이 자동으로 보장된다.

📋
폼 내부에서 제출/초기화할 때

type="submit" 은 폼 전송, type="reset" 은 폼 초기화. type 미지정 시 form 안에선 기본값이 submit 이다 — 이걸 모르면 예상치 못한 폼 전송이 발생한다.

🔗
페이지 이동이 아닌 액션에서 (링크와 구분)

페이지 이동 → <a href>, 동작 실행 → <button>. 이 구분이 시맨틱 마크업의 핵심이며 스크린리더와 검색엔진이 다르게 처리한다.

</> 기본 문법

html
<!-- type 명시가 기본 (기본값은 submit — 폼 밖에선 button) -->
<button type="button" onclick="handleClick()">클릭</button>

<!-- 폼 제출 -->
<button type="submit">제출</button>

<!-- 비활성 -->
<button type="button" disabled>비활성</button>

<!-- 아이콘 포함 (aria-label로 접근성 보장) -->
<button type="button" aria-label="항목 삭제">
  <svg aria-hidden="true">...</svg>
</button>
type 속성 정리
/* type 값과 동작 */
"button"  → JS 이벤트만 실행 (폼 전송 없음) ← 가장 많이 쓴다
"submit"  → 소속된 <form> 전송
"reset"   → 소속된 <form> 초기화

▶ 라이브 데모

button variants — 실무에서 쓰는 패턴
Primary / Secondary / Ghost
Danger / Disabled
Icon + Text (대시보드 액션 버튼)

✕ 자주 하는 실수

✕ <div>로 버튼 만들기
<div class="btn" onclick="submit()">제출</div>
✓ <button> 사용
<button type="button" onclick="submit()">제출</button>
⚠️ <div>는 기본적으로 키보드 포커스가 안 된다. <button>은 Enter/Space 키 동작, Tab 포커스, 스크린리더 읽기가 모두 자동으로 지원된다. 접근성 + SEO 동시 해결.
✕ form 밖에서 type 미지정
<form>
  <input type="text">
  <button>검색</button>  <!-- type 없음! -->
</form>
✓ type 명시
<form>
  <input type="text">
  <button type="button">검색</button>  <!-- 의도한 동작 -->
</form>
⚠️ <form> 안에서 type 미지정 button의 기본값은 "submit". 클릭 시 페이지가 새로고침되거나 예상치 못한 폼 전송이 일어난다. type은 항상 명시해라.
