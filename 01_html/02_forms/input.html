● HTML / Forms

<input>

사용자로부터 데이터를 입력받는 요소 — type에 따라 동작이 완전히 달라진다

✦언제 쓰는가

🔍
검색창, 필터, 텍스트 입력

대시보드 상단 검색, 테이블 필터링 입력창. type="text" 또는 type="search" 사용. search 타입은 브라우저가 X 버튼을 자동으로 제공한다.

📅
날짜/시간 선택, 숫자 범위

type="date", type="number", type="range". 브라우저 기본 UI를 활용하면 커스텀 없이도 실용적인 입력 컴포넌트가 된다.

☑️
체크박스, 라디오 — 단일/다중 선택

테이블 행 선택(checkbox), 옵션 그룹 선택(radio). <label> 과 반드시 연결해야 클릭 영역이 확장되고 접근성이 보장된다.

</>기본 문법

html — 필수 패턴
<!-- label + input 세트: 항상 같이 써라 -->
<label for="email">이메일</label>
<input
  type="email"
  id="email"
  name="email"
  placeholder="example@company.com"
  required
  autocomplete="email"
>

<!-- 숫자 범위 제한 -->
<input type="number" min="0" max="100" step="1" value="0">

<!-- 체크박스 -->
<input type="checkbox" id="agree" name="agree">
<label for="agree">이용약관 동의</label>
자주 쓰는 type 값
"text"     → 기본 텍스트
"email"    → 이메일 형식 검증 자동
"password" → 입력값 숨김
"number"   → 숫자만 (min/max/step 가능)
"date"     → 날짜 선택 UI
"search"   → 검색 (X 버튼 자동)
"checkbox" → 다중 선택
"radio"    → 단일 선택 (같은 name 그룹)
"file"     → 파일 업로드
"hidden"   → 숨김 값 전송 (UI 없음)

▶라이브 데모

input 상태별 스타일 패턴
⚠ 올바른 이메일 형식이 아닙니다
✓ 사용 가능한 아이디입니다

자주 쓰는 type 미리보기

type="date"
type="number"
type="range"
type="checkbox"

✕자주 하는 실수

✕ label 없이 placeholder만
<input type="text" placeholder="이름 입력">
✓ label로 연결
<label for="name">이름</label>
<input type="text" id="name" placeholder="홍길동">
⚠️ placeholder는 입력 시 사라진다. 스크린리더는 placeholder를 label로 인식하지 않는다. for-id 연결로 클릭 영역도 넓어진다.
