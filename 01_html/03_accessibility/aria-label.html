● HTML / Accessibility

aria-label

스크린리더가 읽을 텍스트를 요소에 직접 지정하는 속성

✦언제 쓰는가

🔘
아이콘만 있는 버튼 (텍스트 없음)

× 닫기 버튼, 🔍 검색 버튼, ✏️ 편집 버튼처럼 아이콘만 표시될 때. 스크린리더는 아이콘을 읽지 못하기 때문에 aria-label로 동작을 설명해야 한다.

🗂️
같은 텍스트가 반복될 때 구별

테이블에서 "삭제" 버튼이 여러 행에 있을 때. 스크린리더 사용자는 "삭제, 삭제, 삭제"만 들리면 어느 행인지 모른다. aria-label="홍길동 행 삭제"처럼 맥락을 추가한다.

📊
랜드마크(landmark) 구분

한 페이지에 <nav>가 2개 이상일 때. aria-label="주 메뉴", aria-label="사이드바 메뉴"로 구별한다.

</>기본 문법

html
<!-- 아이콘 버튼 -->
<button type="button" aria-label="모달 닫기">
  <svg aria-hidden="true">...</svg>  <!-- svg는 aria-hidden -->
</button>

<!-- 반복 버튼에 맥락 추가 -->
<button type="button" aria-label="홍길동 행 삭제">삭제</button>

<!-- nav 랜드마크 구분 -->
<nav aria-label="주 메뉴">...</nav>
<nav aria-label="페이지 내 목차">...</nav>
aria 속성 3개 비교
aria-label      → 직접 텍스트 지정 (요소에 인접한 설명 없을 때)
aria-labelledby → 다른 요소의 id를 참조 (이미 텍스트가 DOM에 있을 때)
aria-describedby→ 보충 설명 연결 (에러 메시지, 힌트)

/* 예시 — labelledby 사용 */
<h2 id="modal-title">설정</h2>
<dialog aria-labelledby="modal-title">...</dialog>

▶라이브 데모

아이콘 버튼 — Tab 키로 포커스 이동해봐라

각 버튼에 Tab으로 이동하면 브라우저 접근성 트리에서 aria-label 텍스트가 읽힌다.

aria-label="검색"
aria-label="설정 열기"
aria-label="알림 3개"
aria-label="모달 닫기"

✕자주 하는 실수

✕ SVG에 aria-label 달고 aria-hidden 빠뜨림
<button aria-label="닫기">
  <svg>...</svg>  <!-- aria-hidden 없음 -->
</button>
✓ SVG는 aria-hidden="true" 추가
<button aria-label="닫기">
  <svg aria-hidden="true" focusable="false">...</svg>
</button>
⚠️ aria-hidden 없으면 스크린리더가 버튼의 aria-label과 SVG 내부를 중복으로 읽는다. SVG는 항상 aria-hidden="true"로 숨겨라.
