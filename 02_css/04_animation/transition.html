transition

CSS 속성 변화를 부드럽게 이어주는 애니메이션 처리

✦언제 쓰는가

🖱️
버튼/링크/카드 hover 효과

색상 변화, 그림자, 위치 이동을 부드럽게. transition 없으면 즉각 변화가 어색하고 조잡해 보인다. 150~300ms 범위가 실무 표준.

🎛️
UI 상태 전환 (열기/닫기, 활성/비활성)

드롭다운 노출, 탭 전환, 토글 스위치. 상태 변화를 시각적으로 명확히 전달해 사용자가 무슨 일이 일어났는지 인식하게 한다.

📊
대시보드 데이터 갱신 피드백

숫자가 업데이트될 때, 로딩 후 콘텐츠가 나타날 때. opacity 또는 transform transition으로 자연스러운 등장 효과를 준다.

</>기본 문법

css
/* ── 기본 형태 ── */
transition: 속성명 시간 타이밍함수 딜레이;

/* ── 실무 패턴 ── */
.btn {
  background: steelblue;
  /* 단일 속성 */
  transition: background-color 200ms ease;
}
.btn:hover { background: royalblue; }

/* ── 다중 속성 (성능상 권장) ── */
.card {
  transition:
    transform   200ms ease,
    box-shadow  200ms ease,
    border-color 150ms ease;
}

/* ── 타이밍 함수 ── */
ease               → 기본 (느리게 시작→빠르게→느리게 끝)
ease-in            → 느리게 시작 (퇴장 효과)
ease-out           → 빠르게 시작 후 감속 (등장 효과)
linear             → 일정 속도 (로딩 바)
cubic-bezier(x,x,x,x) → 커스텀 (Material Design 등)

/* ── GPU 가속 속성 (이것만 써라) ── */
transform   → translate, scale, rotate (성능 최고)
opacity     → 투명도 (성능 최고)
/* ── 피해야 할 속성 ── */
width, height, top, left  → 리플로우 유발, 성능 나쁨

✕자주 하는 실수

✕ transition: all 남발
.card {
  transition: all 300ms ease;  /* width, height 포함 → 성능 저하 */
}
✓ 속성 명시
.card {
  transition:
    transform    200ms ease,   /* GPU 가속 */
    box-shadow   200ms ease,
    opacity      200ms ease;
}
⚠️ `all`은 width, height, top, left 등 리플로우를 유발하는 속성까지 포함된다. transform과 opacity만 transition하면 GPU 합성 레이어로 처리되어 60fps 보장.
