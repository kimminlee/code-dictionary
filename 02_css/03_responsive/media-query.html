@media query

화면 크기/조건에 따라 CSS 스타일을 분기하는 규칙

✦언제 쓰는가

📱
데스크탑 레이아웃을 모바일로 변환

3열 카드 → 1열, 사이드바 노출 → 숨김, 폰트 크기 축소. mobile-first(min-width) 방식이 현재 실무 표준이다.

🎨
다크모드 대응

@media (prefers-color-scheme: dark). OS 설정을 감지해 자동으로 다크모드 스타일을 적용한다. CSS 변수와 함께 쓰면 효율적.

🖨️
인쇄용 스타일 분리

@media print. 대시보드 리포트를 PDF로 출력할 때 네비게이션, 버튼 등 불필요한 UI를 숨기는 용도.

</>기본 문법

css — mobile-first 방식 (현재 표준)
/* ── Mobile First: 모바일 기본 → 큰 화면으로 확장 ── */
.grid {
  display:               grid;
  grid-template-columns: 1fr;       /* 기본: 1열 (모바일) */
  gap:                   16px;
}

/* 태블릿 이상 */
@media (min-width: 768px) {
  .grid { grid-template-columns: repeat(2, 1fr); }
}

/* 데스크탑 이상 */
@media (min-width: 1200px) {
  .grid { grid-template-columns: repeat(3, 1fr); }
}

/* ── 다크모드 ── */
@media (prefers-color-scheme: dark) {
  :root {
    --bg:   #0d1117;
    --text: #e6edf3;
  }
}

/* ── 인쇄 ── */
@media print {
  .sidebar, .header, .btn { display: none; }
  body { font-size: 12pt; color: #000; }
}

▶라이브 데모

현재 브레이크포인트 + 실무 기준점
현재 뷰포트: — (—)
📱 Mobile ~767px 1열, 햄버거 메뉴, 스택 레이아웃
🖥️ Tablet 768px ~ 2열, 사이드바 일부 노출
🖥️ Desktop 1024px ~ 3열, 전체 사이드바, 풀 레이아웃
🖥️ Wide 1440px ~ max-width 제한, 콘텐츠 중앙 정렬

✕자주 하는 실수

✕ desktop-first (max-width) 방식
/* 데스크탑 기준 → 모바일로 덮어쓰기 */
.grid { grid-template-columns: repeat(3,1fr); }

@media (max-width: 768px) {
  .grid { grid-template-columns: 1fr; }  /* 덮어쓰기 누적 */
}
✓ mobile-first (min-width) 방식
/* 모바일 기준 → 데스크탑으로 확장 */
.grid { grid-template-columns: 1fr; }  /* 모바일 기본 */

@media (min-width: 768px) {
  .grid { grid-template-columns: repeat(3,1fr); }
}
⚠️ mobile-first는 모바일 성능이 기본값이라 불필요한 CSS 로딩이 없다. Tailwind, Bootstrap 모두 이 방식. 실무 표준.
