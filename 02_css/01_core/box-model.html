box-model

모든 HTML 요소가 갖는 4겹 구조 — content / padding / border / margin

✦언제 쓰는가

📐
요소 크기가 생각과 다를 때 디버깅

width: 200px 으로 지정했는데 실제 렌더 크기가 다르다면 box-sizing 문제다. 크롬 DevTools → Elements → Computed 탭에서 박스모델을 시각적으로 확인한다.

🃏
카드/버튼 내부 여백 설정

클릭 영역을 넓히려면 margin이 아닌 padding. 카드 간격은 padding이 아닌 margin 또는 gap. 이 구분이 레이아웃 설계의 기본.

📏
box-sizing: border-box 전역 적용

width를 지정하면 padding, border를 포함한 크기가 되도록 설정. 실무 프로젝트 reset.css 1번 줄에 들어가는 내용이다.

</>기본 문법

css
/* ── 박스모델 4겹 ── */
.box {
  width:   200px;      /* content 영역 너비 */
  padding: 16px;      /* 내부 여백 (배경색 포함) */
  border:  1px solid #ccc; /* 테두리 */
  margin:  24px;      /* 외부 여백 (투명) */
}

/* ── box-sizing: 실무 필수 전역 설정 ── */
*, *::before, *::after {
  box-sizing: border-box; /* width에 padding+border 포함 */
}

/* ── 단축 속성 ── */
padding: 10px 20px;         /* 상하10 좌우20 */
padding: 10px 20px 15px 5px; /* 상 우 하 좌 (시계방향) */
margin:  0 auto;            /* 수평 중앙 정렬 */

box-sizing 비교 — 둘 다 width: 140px, padding: 16px 설정

content-box (기본값)
내용
실제 렌더 너비: 140 + 32(pad) + 4(border) = 176px
border-box (실무 표준)
내용
실제 렌더 너비: padding 포함 140px

✕자주 하는 실수

✕ box-sizing 없이 width 계산
.card {
  width:   300px;
  padding: 20px;   /* 실제 너비: 340px — 레이아웃 깨짐 */
}
✓ reset에 border-box 전역 적용
*, *::before, *::after { box-sizing: border-box; }

.card {
  width:   300px;  /* padding 포함 정확히 300px */
  padding: 20px;
}
⚠️ border-box를 전역 적용하면 width가 "겉으로 보이는 크기"와 일치한다. 모든 현대 CSS 리셋의 1번 항목이다.
