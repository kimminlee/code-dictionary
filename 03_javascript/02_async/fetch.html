fetch / async-await

서버 API와 비동기로 데이터를 주고받는 표준 방법

✦언제 쓰는가

📡
REST API 호출 — GET/POST/PUT/DELETE

대시보드 데이터 로딩, 폼 제출, 항목 수정/삭제. 모든 프론트엔드에서 API 통신은 fetch 또는 axios로 처리한다.

⏳
순차 실행이 필요한 로직

로그인 → 토큰 저장 → 사용자 정보 조회처럼 앞 결과가 뒤에 필요한 경우. async/await 없으면 Promise 체이닝이 중첩되어 읽기 어렵다.

⚡
여러 API 병렬 호출

Promise.all()로 순서 상관없는 API를 동시에 요청. 순차 호출보다 전체 응답 시간이 단축된다.

</>기본 문법

js — 실무 필수 패턴
// ── 기본 GET ──
async function fetchUsers() {
  try {
    const res = await fetch('/api/users');
    if (!res.ok) throw new Error(`HTTP ${res.status}`); // 필수!
    return await res.json();
  } catch (err) {
    console.error(err);
    throw err;
  }
}

// ── POST ──
async function createUser(body) {
  const res = await fetch('/api/users', {
    method:  'POST',
    headers: { 'Content-Type': 'application/json' },
    body:    JSON.stringify(body),
  });
  return res.json();
}

// ── 병렬 호출 ──
const [users, orders] = await Promise.all([
  fetch('/api/users').then(r => r.json()),
  fetch('/api/orders').then(r => r.json()),
]);

✕자주 하는 실수

✕ response.ok 체크 없음
const res  = await fetch('/api/data');
const data = await res.json();  // 404여도 에러 안 남
✓ ok 체크 + try-catch
try {
  const res = await fetch('/api/data');
  if (!res.ok) throw new Error(res.status);
  const data = await res.json();
} catch (e) { /* 처리 */ }
⚠️ fetch는 네트워크 단절만 reject한다. 400, 404, 500은 정상 resolve된다. response.ok를 반드시 체크해야 HTTP 에러를 잡을 수 있다.
