Array Methods

배열 데이터를 변환·필터·집계하는 ES6+ 핵심 메서드

✦언제 쓰는가

📊
API 데이터를 테이블/차트용으로 변환

서버에서 받은 배열 데이터를 화면에 보여줄 형태로 가공. .map()으로 속성 추출, .filter()로 조건 필터링, .sort()로 정렬. 대시보드 데이터 처리의 90%가 이 세 가지다.

🧮
합계/평균 등 집계

.reduce()로 총합, 평균, 그룹핑. 매출 합산, 카테고리별 집계에 사용. for 루프보다 의도가 명확하고 React state와 잘 맞는다.

🔍
검색/자동완성 구현

입력값으로 목록을 필터링. .filter() + .includes() 또는 .find(). 서버 요청 없이 프론트엔드에서 실시간 필터링할 때.

</>기본 문법

js — 실무 패턴 모음
const orders = [
  { id: 1, user: '김철수', amount: 45000, status: '완료' },
  { id: 2, user: '이영희', amount: 12000, status: '취소' },
  { id: 3, user: '박지민', amount: 88000, status: '완료' },
];

// ① map — 데이터 변환 (원본 불변)
const userNames = orders.map(o => o.user);
// → ['김철수', '이영희', '박지민']

// ② filter — 조건 필터링
const completed = orders.filter(o => o.status === '완료');
// → [{ id:1, ...}, { id:3, ...}]

// ③ reduce — 집계
const total = orders.reduce((sum, o) => sum + o.amount, 0);
// → 145000

// ④ find — 단일 항목 검색
const order = orders.find(o => o.id === 2);
// → { id:2, user:'이영희', ... }

// ⑤ sort — 정렬 (원본 변경 주의!)
const sorted = [...orders].sort((a, b) => b.amount - a.amount);
// → 금액 내림차순

// ⑥ 메서드 체이닝 — 실무 핵심 패턴
const result = orders
  .filter(o => o.status === '완료')   // 완료만
  .map(o => ({ ...o, vat: o.amount * 0.1 })) // VAT 추가
  .sort((a, b) => b.amount - a.amount); // 금액 정렬

✕자주 하는 실수

✕ sort로 원본 배열 변경
// state 배열을 직접 sort — React에서 리렌더 안 됨
const sorted = orders.sort((a,b) => a.amount - b.amount);
✓ 스프레드로 복사 후 sort
// 원본 유지 → React state에 안전
const sorted = [...orders].sort((a,b) => a.amount - b.amount);
⚠️ .sort()는 원본 배열을 직접 변경(mutate)한다. React에서 state 배열을 직접 수정하면 참조가 같아 리렌더가 발생하지 않는다. 항상 복사본을 만들어라.
