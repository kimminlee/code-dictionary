useState

컴포넌트가 기억해야 하는 데이터를 선언하고 변경 시 리렌더를 트리거하는 훅

✦언제 쓰는가

🔘
UI 상태 — 열림/닫힘, 활성/비활성

모달 open/close, 탭 선택 index, 드롭다운 열림 여부. 화면에 보이는 상태는 대부분 useState로 관리한다.

📝
폼 입력값 추적

input의 value를 React가 제어(controlled component). value={state} + onChange={setter} 패턴이 표준.

📊
API 응답 데이터 저장

fetch로 받은 데이터를 state에 저장 → 컴포넌트가 데이터를 렌더링. loading, error 상태도 함께 관리하는 패턴이 실무 표준.

</>기본 문법

jsx
import { useState } from 'react';

// ── 기본 형태 ──
const [state, setState] = useState(초기값);
//     ↑ 읽기     ↑ 변경    ↑ 첫 렌더 값

// ── 실무 예시 1: 모달 open/close ──
const [isOpen, setIsOpen] = useState(false);

return (
  <>
    <button onClick={() => setIsOpen(true)}>열기</button>
    {isOpen && <Modal onClose={() => setIsOpen(false)} />}
  </>
);

// ── 실무 예시 2: API 데이터 + loading + error ──
const [data,    setData]    = useState([]);
const [loading, setLoading] = useState(false);
const [error,   setError]   = useState(null);

async function loadData() {
  setLoading(true);
  try {
    const res = await fetch('/api/users');
    setData(await res.json());
  } catch (e) {
    setError(e.message);
  } finally {
    setLoading(false);  // 성공/실패 무관하게 종료
  }
}

// ── 실무 예시 3: 배열 state 업데이트 ──
const [items, setItems] = useState([]);

// 추가 — 스프레드로 불변성 유지
setItems(prev => [...prev, newItem]);

// 삭제
setItems(prev => prev.filter(i => i.id !== targetId));

// 수정
setItems(prev => prev.map(i => i.id === targetId ? {...i, ...updates} : i));

useState 규칙 (Rules of Hooks)

1
컴포넌트 최상위에서만 호출 if문, 반복문, 중첩 함수 안에서 호출하면 안 된다. React가 훅 순서를 기억하기 때문.
2
React 함수 컴포넌트 또는 커스텀 훅 안에서만 일반 JS 함수에서 호출 불가.
3
state 직접 변경 금지 — setter 함수 사용 state.push() 금지 → setState([...state, item]) 사용

✕자주 하는 실수

✕ 배열 state 직접 변경
const [items, setItems] = useState([]);

// 직접 push → 참조 동일 → 리렌더 안 됨!
items.push(newItem);
setItems(items);
✓ 새 배열 생성
// 스프레드로 새 배열 → 참조 변경 → 리렌더 정상
setItems(prev => [...prev, newItem]);
⚠️ React는 state 참조값을 비교해 리렌더 여부를 결정한다. push()는 원본 배열을 변경해 참조가 같아 리렌더가 발생하지 않는다.
